Privacy Policy

Effective date: January 1, 2025  ·  Last updated: April 6, 2026

Overview

SDR Copilot is a Chrome extension that assists sales development representatives by capturing tab audio during calls, transcribing conversations in real time, and integrating with Gmail and Salesforce to streamline prospecting and CRM workflows.

Because SDR Copilot handles sensitive data including audio, email, and CRM records, this Privacy Policy is detailed. Please read it fully before installing and using the extension.

By installing and using SDR Copilot, you agree to this policy. If you do not agree, do not install the extension.

Data We Collect and Process

CategoryWhat we processHow it's handled
Tab audio Audio from the active browser tab (e.g., a sales call in Orum) Captured via Chrome's tab capture API and streamed in real time to Deepgram for transcription. Audio is not stored on any servers after the session ends.
Call transcripts Text transcription of call audio generated by Deepgram Stored locally in Chrome storage (chrome.storage.local) on your device. Accessible only to you.
Gmail data Email composition access via the Gmail API Used solely to create follow-up email drafts. Not stored beyond the draft creation. You control sending.
Salesforce data Task creation via the Salesforce REST API Written when you explicitly log a call. Data is sent directly from your browser to your Salesforce instance — no intermediary backend.
API credentials Deepgram API key, Salesforce OAuth tokens, Google OAuth tokens Stored in chrome.storage.sync (Google-encrypted, synced across your signed-in Chrome devices). Never transmitted to SDR Copilot servers.
Call metadata Duration, objection count, disposition, prospect name Stored locally in chrome.storage.local. Up to 500 calls retained. Never sent to external servers.
Audio capture disclosure: SDR Copilot captures audio from browser tabs when you explicitly activate recording. You are responsible for ensuring that all parties on a call have been informed and have consented to recording, as required by applicable wiretapping and call recording laws in your jurisdiction. SDR Copilot does not activate recording without your explicit action.

Audio Data and Transcription

When you start a recording session, audio from the active tab is streamed in real time to Deepgram for speech-to-text transcription. The following applies:

  • Audio streams are sent over encrypted connections (TLS/WSS)
  • Deepgram processes audio to generate transcripts and does not use API audio for model training by default
  • Raw audio is not stored on Deepgram's servers after the transcription session ends (subject to Deepgram's current data retention policy)
  • Transcripts are stored locally on your device in Chrome storage, not on any SDR Copilot server
  • You can delete transcripts from the dashboard at any time

See Deepgram's privacy policy at deepgram.com/privacy.

Gmail API Access

SDR Copilot requests Gmail compose access to draft post-call follow-up emails. Our use of data obtained from Gmail APIs complies with the Google API Services User Data Policy, including the Limited Use requirements.

  • We access only the Gmail compose scope — sufficient to create drafts
  • We do not read your existing email, contacts, or inbox
  • We do not use Gmail data for advertising or profile-building
  • We do not share Gmail data with any third party
  • You can revoke Gmail access at any time via your Google Account → Security → Third-party apps

Salesforce API Access

SDR Copilot connects to Salesforce using OAuth2 to create Task records after calls. All Salesforce API calls are made directly from your browser to your Salesforce instance — there is no SDR Copilot backend server that stores or proxies your Salesforce data.

We do not read, store, or analyze your Salesforce CRM records beyond what is necessary to create the post-call Task.

Chrome Extension Permissions

SDR Copilot requires the following Chrome permissions:

  • tabCapture: To capture audio from the active tab when you start a recording
  • tabs: To identify the active tab and detect Orum sessions
  • storage: To persist settings, API keys, and call history locally
  • identity: To handle OAuth authentication with Google and Salesforce
  • offscreen: Required by Chrome MV3 for audio processing without a visible tab
  • notifications: To display post-call summary notifications

The extension does not use camera, clipboard (read), or browsing history permissions.

Third-Party Services

  • Deepgram: Real-time speech transcription. Audio is streamed during active recording sessions only. See deepgram.com/privacy.
  • Google (Gmail API + Chrome Identity OAuth): Gmail draft creation and authentication. See policies.google.com/privacy.
  • Salesforce: CRM Task creation. Subject to your organization's Salesforce data processing agreements. See salesforce.com/company/privacy.

Data Storage and Security

  • All data transmissions use TLS/WSS encrypted connections
  • API credentials are stored in chrome.storage.sync, which is encrypted by Google
  • Call history and transcripts are stored in chrome.storage.local on your device only
  • There is no SDR Copilot cloud backend that stores your data — this is a client-side extension

Data Retention

Call history is stored locally up to 500 calls (oldest entries are removed automatically). You can clear all local data at any time from the extension's settings. Removing the extension from Chrome also removes all locally stored data.

Your Rights

  • Access: All your data is stored locally on your device and is directly accessible in the dashboard
  • Deletion: Delete individual calls from the dashboard, or clear all data via Chrome → Extensions → SDR Copilot → Storage → Clear
  • Revoke integrations: Revoke Gmail access via Google Account settings; revoke Salesforce access via Salesforce Connected Apps

To contact us about privacy: privacy@sdrcopilot.com

Call Recording Compliance

Call recording laws vary by jurisdiction. In some U.S. states and many countries, all-party consent is required before recording a phone or online call. You are solely responsible for complying with applicable laws when using SDR Copilot's recording features.

Best practice: Always inform all call participants that the call is being recorded, regardless of jurisdiction.

Children's Privacy

SDR Copilot is a professional tool intended for use by adults in sales roles. We do not knowingly collect data from persons under 18 years of age.
